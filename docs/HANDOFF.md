# RobikServer — Handoff / стан проєкту (станом на 2026-07-01)

> Встав цей файл (або його ключову частину) на початку нового чату, щоб одразу дати контекст.

## Що це
RobikServer (`F:\prog\RobikServer`) — Twitch-бот/платформа для власних стрімів. Монорепо:
- **apps/server** — Fastify + Prisma (PostgreSQL, adapter-pg) + Twitch EventSub (WebSocket). Запуск: `npm run dev` (tsx watch). Порт 4000.
- **apps/admin** — React + Vite адмін-панель.

Фаза: **chat-only** фічі. Інтеграція Unity/Unreal свідомо відкладена (є WS-сервери, але економіка/ігри поки лише в чаті).

## Головний принцип (не порушувати)
**Усі назви команд і числа — конфігуровні через БД** (`EconomySettings`, `GiveawaySettings`, `GuessGameSettings`, fun-meter features) + **усі повідомлення бота редаговані** в адмінці. Жодних хардкод-назв команд.

## Архітектура (де що)
- Команди чату йдуть через `modules/twitch/handlers/ChatMessageHandler.ts` → ланцюг роутерів. Порядок: touch/stats → award chat-activity → funMeter → roulette → steal → fight → guess → economy(!баланс/!топ/!передати) → status(!профіль) → help(!команди) → giveaway → custom.
- DI-контейнер: `modules/twitch/TwitchRuntimeContainer.ts` (усе інстанціюється тут).
- Налаштування економіки: `economy/EconomyService.ts` (кеш 30с) + `EconomyRepository.ts` + `economy.types.ts` + `economy.messages.ts`. Зміна поля = 5 місць: schema.prisma → міграція → economy.types DTO → EconomyService(normalize+toSettingsDto) → EconomyRepository.updateSettings (+ admin types/economy.ts + EconomyPage).
- Гроші: `EconomyService.award/spend/transfer` (atomic, anti-overdraw). Гаманець = `ViewerWallet` (unique `twitchUserId`). `sim:`-гаманці перехоплюються в `EconomyRepository` → in-memory `SimWalletStore`.
- Стан рантайму/лайв: `runtime/TwitchRuntimeState.ts`.
- Авторизація адмінки: `modules/auth/` (`AdminSessionService` — HMAC-сесія; `admin-auth.guard.ts` — глобальний guard + список публічних шляхів; `AuthController`/`auth.routes.ts` — `/auth/admin/*`). Twitch-OAuth для самого бота (токени broadcaster/bot) — теж тут (`AuthService`, `TwitchOAuthService`), не плутати з admin-login.
- Довідник команд: `modules/twitch/command-guide/CommandGuideService.ts` (зберігання в `AppSetting` + генерація з налаштувань).
- Admin-фронт: роути в `apps/admin/src/App.tsx` (публічний `/guide` + `RequireAuth` навколо `AdminLayout`); auth-стан у `components/auth/AuthContext.tsx`; HTTP-клієнт `api/http.ts` шле `credentials:include` і на 401 кидає подію `admin-unauthorized`.

## Що ВЖЕ ЗРОБЛЕНО (усе server+admin tsc clean, міграції застосовані)
1. **Економіка/гаманці** — баланс, !топ, !передати, !профіль; заробіток за чат-активність + присутність (Helix Get Chatters poll); люркери −%.
2. **Бафи/дебафи** — рулетка ефектів (!ефект), множники/шанс/flat/guarantee, впливають на роли й заробіток.
3. **Fun-meter** — налаштовувані «міри» (!удача тощо), топ/свій результат, increaseChance.
4. **Рулетка** (`!рулетка`) — переписана: ставки `100`/`all`/`NN%`, виплата **1:1** (виграш додається, програш забирає; payoutPercent), шанс **випадковий у діапазоні** (win-chance min/max), **кулдаун**, **leader-lock** (топ-1 крутить лише all + не передає, поки не виграє all-in; програв → блок до наступного).
5. **Крадіжка** (`!вкрасти`/`!щит`) — штраф = **% балансу злодія** → жертві; **тег жертви + затримка** (stealWarnSeconds); жертва відбивається чатом або щитом → скасування + штраф.
6. **Бійки** (`!бійка @нік <ставка>` + `!приймаю`) — PvP виклик+прийняття, переможець забирає ставку (зеро-сум).
7. **Вгадай число** (`!цифри <від> <до> [таймер]` / `!стопцифри`) — мод/стрімер старт, перший вгадав отримує нагороду.
8. **Розіграш** (емісійний, мод/стрімер, пресети) + **розіграш своїх балів** (`!розіграшсвоїх <сума>` — списує власні бали, рефанд якщо 0 учасників).
9. **!команди** — вбудована динамічна довідка, одне повідомлення; назву/шаблон редагуєш на сторінці «Команди».
10. **Сторінка «Хто на стрімі»** — лог глядачів за сесію + статус «пише/мовчить (люркер)».
11. **Ручне нарахування** — фікс: резолв за **логіном** (наявний гаманець або Helix), не створює дублів.
12. **Бот і стрімер не отримують балів** (виключені із заробітку).
13. **Live-gating (2026-06-30):** пасивне нарахування + статистика глядачів працюють **лише коли стрім у ефірі** (EventSub stream.online/offline + Helix Get Streams на старті; прапорець `streamLive` у runtimeState; видно на сторінці Runtime). Інтерактивні команди НЕ ґейтяться.

21. **Система перків: guest / loyal / supporter (2026-07-01, server+admin tsc + vite build чисті; наживо НЕ тестовано):** градуйовані рівні глядачів, кожен наступний включає перки попереднього, але сильніші. Новий модуль `modules/twitch/supporter/`. **Рівні:** 🟢 `guest` (усі); 🔵 `loyal` — **безкоштовно** за presence-streak (деф. 5 днів поспіль, конфіг; **згорає при пропуску стріму**); 🟣 `supporter` — **ручний список в адмінці** (заглушка; згодом mono-підписка через webhook писатиме `monoSubId`/`monoUntil`). Резолв: `SupporterService.resolveTier(login)` = supporter(manual/mono) → loyal(streak≥поріг) → guest. **Принцип «тільки чесно»:** множники/бонуси/косметика — так; **жодного pay-to-win** у PvP-іграх (steal/рулетка/бійки однакові для всіх). **Моделі:** `SupporterSettings` (усі числа перків + повідомлення, ключ `supporter`) + `SupporterStatus` (ключ `userLogin`; `manualTier`/`manualUntil` = ручний список, `monoSubId`/`monoUntil` = фаза 2, `streakDays`/`lastStreamDay` = loyal, `lastBonusAt` = кулдаун !бонус). Ланцюг стрім-днів у `AppSetting` (`supporter_streak_chain`). Міграції: `add_supporter_perks`, `add_supporter_bonus_claim`. **Що вже впливає на бота:** (а) **множник валюти** за рівнем (loyal ×1.5, supporter ×2.5) — у `ChatMessageHandler.awardChatActivity` + `PresenceEarningService.tick` (перемножується з баф-множниками; **тільки заробіток**, не виграші в іграх); (б) **presence-streak** росте раз/стрім-день у `PresenceEarningService.tick` (live-gated), згорає на початку наступного стріму після пропуску (`resetLapsedStreaks`); (в) **`!бонус`** (`SupporterBonusCommandRouter`, назва з налаштувань) = `щоденний бонус(рівень) + стрік-бонус(якщо streak>0)`, кулдаун персистентний (`lastBonusAt`, деф. добовий), НЕ live-gated, джерело `"supporter.bonus"` у `EconomyChangeSource`; (г) **пріоритетна пісня** — supporter `!пісня` → `priority=supporterSongPriority` (як за донат); (д) **привітання** loyal/supporter при першому повідомленні за день (`ChatMessageHandler.maybeGreet`, live-gated + виключає бота/стрімера; несе бейдж 🔵/🟣). Бейдж на КОЖНУ відповідь свідомо НЕ робили (крихко) — привітання = носій бейджа. **Роути** (`twitch.routes.ts`, під admin-guard): `GET/PATCH /twitch/supporter/settings`, `GET/POST/DELETE /twitch/supporter/subscribers[/:login]`, `GET /twitch/supporter/inspect/:login`, `POST /twitch/supporter/debug/{streak,reset-bonus}`. **Адмінка:** сторінка «Перки» (`SupporterPage`, вкладки Рівні|Підписники|Повідомлення), пункт сайдбару «Перки» (♛). **Тест-панель у «Симуляції»** (`PerkTester.tsx`): inspect рівня, зробити підписником / виставити стрік / скинути кулдаун, прогнати `!бонус`/`!пісня` на персистентному sim-юзері (результат у «Результатах»). ## Статус / що НЕ зроблено
- **Вхід через Twitch працює** (перевірено користувачем). Публічний довідник + редагування — server+admin `tsc` чисті, ізольовані smoke-тести guard пройдено, але редагування/генерацію довідника в живій адмінці ще не клікали.
- **Runtime НЕ протестовано наживо** для economy v2 + live-gating — повний бут із БД/Twitch на справжньому стрімі ще НЕ ганяли (інтерактив/команди в чаті не перевірені на живому EventSub). `.env` уже має заповнений `ADMIN_ALLOWED_LOGINS` і згенерований `ADMIN_SESSION_SECRET`.
- Багато in-memory станів (leader-lock cleared set, pending steal/fight) — **скидаються при рестарті процесу** (за дизайном). (Виняток: presence-лог «Хто на стрімі» тепер персистентний у БД — див. п.19.)
- **Симулятор чату** (сторінка «Симуляція») проходить live-gate → офлайн sim не нараховує/не пише статистику; відкладені таймери крадіжки/бійки/розіграшу спрацьовують після захопленого контексту (повний результат у симуляторі не видно). **Sim-гаманці (`sim:`) тримаються в памʼяті** (`SimWalletStore`) — у БД/основний список не потрапляють, скидаються при рестарті процесу.
- Економічні числа (ставки заробітку, ціни) — дефолти підібрані, але **тонке балансування ще можливе** (рулетка тепер нетто-сток — головний фікс інфляції). Гаманці були **очищені** раніше — чистий старт.
- **Система перків (п.21)** — код чистий, але наживо не ганяли: множник заробітку + привітання спрацьовують лише в ефірі (live-gate), тому їх у дії видно тільки на реальному стрімі; `!бонус` + пріоритет пісні перевіряються офлайн через тест-панель у «Симуляції». **Платний рівень supporter через mono acquiring** («Оплати на сайті») — чекає верифікації еквайрингу; наразі supporter додається **вручну** в адмінці. Довідник mono acquiring — у памʼяті `robikserver-song-request`.
- Не зроблено (за бажанням): кнопка «нарахувати» в рядку таблиці гаманців; обхід live-gate для симулятора; realtime-SSE для деяких сторінок.
